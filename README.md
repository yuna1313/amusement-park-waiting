# 놀이공원 대기 프로그램

콘솔에서 동작하는 놀이공원 대기열 시뮬레이션 프로그램입니다. 사용자가 직접 줄을 서지 않고, 놀이기구별 대기 현황을 확인하고 원하는 놀이기구에 대기를 등록할 수 있습니다. 프로그램은 0초부터 시작하며, 메뉴를 선택할 때마다 시간이 1초씩 흐릅니다.

## 프로젝트 개요

- 사용자 등록(이름 + 티켓 타입) 후 메인 메뉴를 통해 기능을 이용합니다.
- 시간이 흐르면 자동으로 대기자가 늘어나고, 놀이기구는 정해진 주기마다 운행되어 대기자를 태웁니다.
- FastPass 티켓으로 일반 줄을 건너뛰어 우선 탑승할 수 있습니다.

### 놀이기구 구성

| 놀이기구 | 운행 주기 | 1회 수용 인원 |
|---|---|---|
| 스톤 익스프레스 | 5초 | 4명 |
| 아르카나 라이드 | 7초 | 6명 |
| 에오스 타워 | 7초 | 8명 |
| 자이로 스핀 메이플 리뉴얼 버전 | 7초 | 10명 |

### 티켓 타입

| 입력 | 티켓 | FastPass 횟수 |
|---|---|---|
| 1 | 일반 | 0 |
| 2 | FastPass 1회 | 1 |
| 3 | FastPass 3회 | 3 |

## 사용 방법

프로그램을 실행하면 먼저 **사용자 등록**(이름 입력 → 티켓 타입 선택)을 진행한 뒤 메인 메뉴가 나타납니다.

| 번호 | 메뉴 | 설명 |
|---|---|---|
| 1 | 놀이기구 대기 현황 조회 | 전체 놀이기구의 대기 인원·예상 시간, 내 대기 위치/남은 시간 확인 |
| 2 | 놀이기구 줄서기 | 놀이기구를 골라 일반/FastPass 대기열에 등록 |
| 3 | 나의 탑승 내역 조회 | 실제 탑승 완료된 기록 확인 |
| 4 | 인기 놀이기구 조회 | 누적 대기 인원 기준 TOP 3 |
| 5 | 나의 정보 조회 | 이름·티켓 타입·FastPass 잔여 횟수 |
| 0 | 종료 | 프로그램 종료 |

### 대기 → 탑승 흐름

1. **줄서기**: 놀이기구 선택(목록에서 `0` 입력 시 뒤로 가기) → (FastPass 사용 가능 시) Y/N 선택 → 대기 등록
2. **이동 안내(입장 15초 전)**: 놀이기구로 이동하라는 알림 출력
3. **도착 확인(입장 5초 전)**: 도착 여부를 Y/N로 입력
   - `Y`: 탑승 의사 확정 → 실제 탑승 시간에 탑승 처리
   - `N`: 대기열에서 제거(탑승 내역에 기록되지 않음)
4. **탑승 처리**: 실제 탑승 시간이 되면 탑승 완료. FastPass로 탑승 시 잔여 횟수 1회 차감

## 특이사항 / 동작 규칙

- **시간 진행**: 메뉴를 선택할 때마다 1초씩 증가합니다.
- **자동 대기자**: 시작 시 각 놀이기구에 5명, 이후 5초마다 자동으로 대기자가 추가됩니다. 일반 대기는 **수용 인원에 비례**(수용 ÷ 2)해서, FastPass는 1명씩 추가됩니다. (예: 수용 4 → 일반 2명, 수용 10 → 일반 5명)
  - **대기열 상한**(일반 20명 / FastPass 10명)을 두어 대기 인원이 무한정 늘어나지 않습니다.
- **예상 대기 시간**: 대기 현황의 시간은 "**지금 줄을 서면** 걸리는 시간"을 뜻합니다. 내가 이미 대기 중이면 **탑승까지 남은 시간**이 함께 표시됩니다.
- **FastPass 규칙**
  - FastPass는 **동시에 1개**만 등록할 수 있습니다. 이미 FastPass로 대기 중이면 다른 기구는 일반 대기열로 등록됩니다.
  - 평소에는 FastPass가 일반 줄보다 **우선 탑승**합니다. 그래서 일반 대기 중에는 FastPass 대기자에게 밀려 탑승이 늦어질 수 있습니다(실제 놀이공원의 우선 탑승과 동일).
  - FastPass 대기 인원이 수용 인원을 초과하면 좌석을 나눠 태웁니다. 이때 FastPass에 더 많은 좌석을 배정하되(약 2:1), **일반에 최소 1석**은 보장합니다. (예: 수용 6명 → FastPass 4 / 일반 2(2:1), 수용 10명 → FastPass 7 / 일반 3, 수용 4·8명 → 약 3:1)
- **동시 대기 제한**: 한 번에 **일반 대기 1개 + FastPass 대기 1개**까지만 가질 수 있습니다. 같은 놀이기구에 중복 대기는 불가합니다.
- **인기 순위**: **누적 대기 인원**(그 놀이기구에 등록된 모든 대기자의 합, 자동 대기자 포함) 기준으로 정렬됩니다. 자동 대기자가 수용 인원에 비례해 추가되므로 수용 인원이 큰 놀이기구일수록 누적 대기 인원이 더 많이 쌓입니다.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `main.py` | 프로그램 진입점 (시스템 생성 후 실행) |
| `amusement_park_system.py` | 전체 시스템 로직(메뉴, 시간 진행, 자동 대기자, 알림 등) |
| `ride.py` | 놀이기구(대기열, 운행/탑승 처리, 예상 시간 계산) |
| `visitor.py` | 사용자(티켓, FastPass 횟수, 탑승 기록) |
| `waiting_info.py` | 대기 정보(사용자·대기 종류·예상 시간·알림 상태 등) |
| `boarding_record.py` | 탑승 기록(놀이기구·시간·대기 시간·대기 종류) |
